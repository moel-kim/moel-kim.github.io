## 읽기 전

블로그에 한글을 사용하는 법을 알아냈습니다.\
앞으로 좋은 영문글은 한국어로 변역해서 올리도록 하겠습니다 :)\
아래의 글은 [Effective Go](https://go.dev/doc/effective_go)의 번역본 입니다

## 개요

Go는 새로운 언어입니다. 기존에 있던 언어들의 아이디어를 차용하였지만,
그런 언어들과 비교했을때 Go 프로그램을 특이하게 만들어주는 요소들이 있습니다.
C++이나 Java 프로그램을 그대로 Go언어로 포팅하는 것은 만족스러운 결과를 얻을 수 없을 것 입니다.
왜냐하면 Java 프로그램은 Java로 짜여져있으니까요. 반대로 말하면, Go를 중심으로 프로그램을 작성한다면
만족스러운 프로그램을 만들 수 있지만 다른 프로그램과 상당히 다른 결과를 얻을 수 있습니다.
다른 말로, Go언어를 잘 하려면 해당 언어가 가지고 있는 특성과 규칙을 알아야합니다.
Go언어를 구성하고 있는 네이밍, 포멧팅, 프로그램 구조를 잘 따르면서 프로그램을 작성한다면
다른 Go 프로그래머들도 쉽게 이해할 수 있는 프로그램을 만들 수 있습니다.

해당 글은 깔끔하고 규칙을 잘 지킨 Go 코드를 작성하는 방법을 소개합니다.
읽기 전 language specification, Tour of Go, How to Write Go Code를 참고하세요.

추가사항(2022.01): 이 글은 2009년 Go언어 출시를 위해 만들어졌으며, 그 이후 추가된 내용은 특별히 없습니다.
Go언어의 안정성 덕분에 해당 글은 여전히 언어 자체를 이해하는데에는 좋은 글입니다.
그러나 빌드 시스템, 테스팅, 모듈, 다형성과 같은 많은 변화가 일어난 부분에 대해서는 설명이 없거나 적습니다.
이 부분에 대해서 업데이트 할 계획은 없으며, 모던 Go의 사용법에 대해서는 수 많은 블로그, 문서, 책들이 있습니다.
Effective Go는 아직 유용하지만 Complete Guide와는 거리가 멀다는 점을 알려드립니다.

## 예시

Go package source들은 core library일 뿐만 아니라 어떻게 사용하는지 알려주고 있습니다.
게다가 많은 패키지들은 golang.org 웹사이트에서 즉시 실행해볼 수 있는 실행가능한 예제들도 포함하고 있습니다.
만약 문제 접근법이나 어떻게 구현되어 있는지에 대해 알고 싶다면, 라이브러리 안에 있는 문서와 코드들을 참고하세요.
