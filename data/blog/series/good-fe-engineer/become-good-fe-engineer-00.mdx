# Intro

좋은 프론트엔드 개발자 되기 시리즈를 연재하려고 합니다.

지난 4년 동안  외주 및 풀타임 프론트엔드 개발자로서 현업에 있었습니다. 지난 시간동안 엔지니어링 적으로 다양한 문제점들을 겪었고 때로는 우아하게, 가끔은 지저분하게 문제를 해결했습니다. 

문제를 이렇게 해결했으면 내 코드가 좀 더 잘 쓰일 수 있지 않을까 반성하면서 시리즈를 연재하려고 합니다.

주로 타입 안전성을 지키면서 빠르게 리팩토링하는 방법을 소개할 예정이며 다음과 같은 주제로도 글을 쓸 예정입니다.

## Core

- 형상관리
	- Make & Skeleton 패턴
- 사이드 이펙트를 위임하는 방법
	- Browser 주소창을 이용한 영속성 위임하기
	- Uncontrolled State를 이용한 영속성 위임하기
	- Funnel 관리하기
* What과 How를 분리하는 방법
	* React Suspense & ErrorBoundary(Making SmartSuspens)
- 데이터 모델링(with. 대수적 데이터 타입)
- [ts-belt](https://mobily.github.io/ts-belt/)로 알아보는 함수형 프로그래밍의 장점과 단점
	- 데이터 파이프라이닝
	- `Option 타입` VS `Result 타입`()
	- Currying과 Partial Application
	- Opaque Type

## Sub

* React-Query key 관리하기
* Mock 서버 생성 자동화하기(with. OpenAPI Spec)
* 동료의 실수를 방지하는 개발자 되기


## Opinion

* 비지니스 고려하기
	* `유지보수가 쉬운 코드` vs `빠르게 실행가능한 코드`
